# Teams Control — Stream Deck Plugin

A Stream Deck plugin that toggles **Microsoft Teams mute and camera** — and it
works **globally**, even when Teams is not the foreground app.

It does this through the **official Microsoft Teams third-party app API**: a
local WebSocket server (`ws://localhost:8124`) that the *new* Teams client
exposes for exactly this kind of integration. This is the supported replacement
for the old, discontinued Microsoft Stream Deck plugin — and because it is a
real API (not a simulated keypress) it does not require Teams to be focused.

## Features

- **Teams Mute** key — toggles your microphone, shows live (green) / muted (red).
- **Teams Camera** key — toggles your camera, shows on (green) / off (red).
- Both keys reflect the real Teams state: if you mute from inside Teams, the
  key updates automatically.
- Outside a meeting the keys show a greyed-out icon and do nothing (the Teams
  API only allows toggling while you are in a meeting/call).

## Requirements

- macOS 12+ (tested target: MacBook Air M4, macOS Sequoia) or Windows 10+.
- **New Microsoft Teams** — the classic Teams client does not expose this API.
- Elgato Stream Deck app **6.5 or newer**.
- Node.js 20 LTS (for building).

## One-time setup

### 1. Enable the Teams API

In the new Teams: **Settings → Privacy → Third-party app API** → turn it **on**.

### 2. Build and install the plugin

```bash
npm install
npm run build
npm install -g @elgato/cli            # if you don't have the Stream Deck CLI
streamdeck link com.kosmonautica.teams-control.sdPlugin
streamdeck restart com.kosmonautica.teams-control
```

### 3. Add the keys and pair

1. In the Stream Deck app, drag **Teams Mute** and/or **Teams Camera** onto a key.
2. Join any Teams meeting (a "Meet now" with yourself is enough to test).
3. The key briefly shows **PAIR** while the plugin negotiates with Teams.
   Pairing happens automatically — no dialog to click. After a few seconds the
   plugin receives a token and the key switches to its normal (greyed) state.
   You only have to do this once; the token is stored and reused on restart.

## How it works

```
Stream Deck key ──► plugin.js (Node) ──► ws://localhost:8124 ──► Microsoft Teams
                         ▲                                            │
                         └──────────── meeting state updates ─────────┘
```

- `src/teams-client.ts` — a single shared WebSocket connection to Teams:
  handles pairing/token persistence, automatic reconnect with backoff, and
  broadcasts meeting-state updates.
- `src/actions/base-action.ts` — shared key behaviour: subscribe to state,
  render the icon, forward presses while in a meeting.
- `src/actions/toggle-mute.ts` / `toggle-camera.ts` — the two concrete actions.

npm run watch    # rebuilds and restarts the plugin on change

## Debugging

### Enable developer mode (required for `streamdeck restart`)

```bash
defaults write com.elgato.StreamDeck developerMode YES
pkill -x "Stream Deck" && open -a "Stream Deck"
```

You only need to do this once. Without it, `streamdeck restart` fails with
"Feature only enabled in developer mode".

### Plugin log file

The new Stream Deck SDK writes logs inside the plugin bundle, **not** in
`~/Library/Logs/`. The correct path:

```
~/Library/Application Support/com.elgato.StreamDeck/Plugins/com.kosmonautica.teams-control.sdPlugin/logs/com.kosmonautica.teams-control.0.log
```

Tail it while you test:

```bash
tail -f ~/Library/Application\ Support/com.elgato.StreamDeck/Plugins/com.kosmonautica.teams-control.sdPlugin/logs/com.kosmonautica.teams-control.0.log
```

The log file is created the moment the plugin loads its first action key. If the
file does not exist yet, add one key to your Stream Deck layout, wait a second,
then check again.

### Reading the log — what to look for

A healthy startup with a previously-paired token looks like:

```
Teams Control plugin starting
Connecting to Teams API (paired: true)
Teams WebSocket open
```

Followed shortly by a `meetingUpdate` JSON blob once you are in a meeting. If
you see these lines in sequence, the plugin is working.

Common failure patterns:

| Log line | Cause | Fix |
|---|---|---|
| `Connecting to Teams API (paired: false)` | No token stored yet, will try to pair | Join a meeting — pairing completes automatically |
| No log file at all | Plugin never loaded | Add a key to the layout; rebuild if still missing |
| Log stops at "Connecting…" with no "open" | Teams API not listening on port 8124 | Enable API in Teams Settings → Privacy; restart Teams |
| `Teams API error: Does not fit protocol standard` | Plugin sent a command Teams doesn't recognise | Update to the latest plugin build |
| `Unparseable Teams message` | Received non-JSON from Teams | Usually harmless; log will show the raw text |
| `Cannot send "toggle-mute": Teams not connected` | Key pressed while socket is closed | Press again — key press triggers immediate reconnect |

### Verify Teams API is listening

```bash
lsof -nP -iTCP:8124 | grep LISTEN
```

If nothing is listed, the Teams API is not running. Check:
1. You are using **new Teams** (not the classic client).
2. **Settings → Privacy → Third-party app API** is enabled.
3. Teams was restarted after enabling the setting.

### Manual WebSocket probe (`tools/ws-test.mjs`)

Use this to test the Teams API directly, without the Stream Deck plugin:

```bash
node tools/ws-test.mjs              # no token — auto-pairs when canPair=true
node tools/ws-test.mjs <token>      # test with a specific saved token
node tools/ws-test.mjs --empty      # reproduces the empty-token bug (Teams silently ignores this)
```

**Important:** Teams only accepts **one** WebSocket client at a time. Running
`ws-test.mjs` will drop the plugin's connection while it runs. Stop it
(`Ctrl-C`) and press a Stream Deck key to reconnect the plugin.

A successful unpaired run looks like:

```
[ws-test] connecting: ws://localhost:8124?protocol-version=2.0.0&…
[ws-test] OPEN
[ws-test] MSG: {"meetingUpdate":{"meetingPermissions":{"canPair":false,…}}}
  … join a meeting …
[ws-test] MSG: {"meetingUpdate":{"meetingPermissions":{"canPair":true,…}}}
[ws-test] sending pair request: {"action":"pair","parameters":{},"requestId":1}
[ws-test] MSG: {"tokenRefresh":"<long token string>"}
```

The `tokenRefresh` line confirms pairing works. The plugin stores this token
automatically when it runs through the same flow.

### Reset pairing (get a fresh token)

If you need to re-pair (e.g. after reinstalling Teams):

```bash
streamdeck unlink com.kosmonautica.teams-control.sdPlugin
streamdeck link com.kosmonautica.teams-control.sdPlugin
streamdeck restart com.kosmonautica.teams-control
```

Then join a Teams meeting — pairing happens automatically within a few seconds.

## Troubleshooting

### Key stays grey and shows no text, even in a meeting

1. **Rebuild and restart.** `bin/plugin.js` is not tracked by git; after every
   `git pull` you must rebuild:
   ```bash
   npm run build
   streamdeck restart com.kosmonautica.teams-control
   ```
2. **Check the plugin log** — see the *Debugging* section above for the exact
   path. Look for `Connecting to Teams API`, `Sent pair request`,
   `Received Teams pairing token`.
3. **Confirm Teams API is enabled and listening:**
   ```bash
   lsof -nP -iTCP:8124 | grep LISTEN
   ```
   Nothing listed → re-check **Settings → Privacy → Third-party app API** in the
   *new* Teams, then restart Teams.
4. **Press the key** to force an immediate reconnect if the backoff timer is active.

### Key shows "PAIR" and stays there

The plugin is connected but waiting for `canPair: true` from Teams. This only
happens while you are in a meeting. Join a meeting — pairing completes
automatically within a few seconds.

### After rebuild, `streamdeck restart` fails

Enable developer mode (see *Debugging* above), then:
```bash
pkill -x "Stream Deck" && open -a "Stream Deck"
```

## Known limitations

- Works only with the **new** Teams client, not classic Teams.
- Mute/camera can only be toggled **while in a meeting or call** — the Teams
  API restricts this, hence the greyed-out state otherwise.
- Pairing happens once per machine; the token is stored in Stream Deck's global
  plugin settings.
